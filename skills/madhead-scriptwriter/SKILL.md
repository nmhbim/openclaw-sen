# Madhead Scriptwriter

Write a long-form military-news YouTube script in the Madhead style.

## Overview

Use this skill when the user wants a dramatic, technical, conflict-focused YouTube script with strong hooks, dense operational details, controlled uncertainty language, and a packaging bundle for upload.

## Workflow

1. Clarify the brief only if needed.
   - Confirm the event, actors, weapons, and scenario.
   - Confirm whether the video should stay strictly factual or may include light hypothetical escalation.

2. Gather current information.
   - Search for the newest public reporting before drafting.
   - Prefer open-source reporting and recent coverage.
   - If certainty is weak, preserve uncertainty in the script.

3. Read the references before writing.
   - Read `references/language-safety.md` for mandatory uncertainty phrasing.
   - Read `references/style-guide.md` for narrator voice and sentence rhythm.
   - Read `references/script-structure.md` for section-by-section requirements.
   - Read `references/packaging-rules.md` before generating titles, description, hashtags, and tags.

4. Write the script body first.
   - Produce a continuous script in prose.
   - Do not use bullet lists inside the script body.
   - Keep the script in the required length range unless the user asks otherwise.

5. Run a safety and style pass.
   - Remove overconfident factual claims.
   - Replace certainty language with approved hedge language where needed.
   - Ensure numbers, weapons, and tactical details are woven naturally into the narration.

6. Deliver the packaging bundle.
   - Provide the script first.
   - Then provide 5 title ideas, 1 YouTube description, 10 SEO hashtags, and 20 extended tags.
   - Follow `assets/output-bundle-template.md`.

## Guardrails

- Use uncertainty language for disputed or fast-moving military reporting.
- Do not present hypothetical escalation as confirmed fact.
- Do not write in a detached analytical essay voice; keep the script cinematic and scene-driven.
- Do not end with a fully closed conclusion; leave a lingering image, lesson, or question.
- Do not output notes, planning commentary, or meta-explanations unless the user asks.
